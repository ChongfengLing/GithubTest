### 分支Branch：

1. __定义__：
    - 独立的2个存放代码的地方。我们一般分成master和dev两个分支。无主次之分，但我们在master中存放稳定版本，在dev中存放开发版本。
2. __性质__：
    - dev中的版本比master中的新。
    - dev的更新比master的频繁。
    - master中的版本比dev的有更少（无）bug。
    - master与dev有相同但独立的push、fetch&merge、add、commit等命令。
    - dev分支是我们日常工作的分支。
3. __定理__：
    - 通过merge进行两个分支见代码的更新合并（把dev的版本merge到master中）。
